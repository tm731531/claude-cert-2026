🧭 策略對照 · Strategy Catalog

同一個情境有一整排有名字的策略——架構師要知道整排,不只自己慣用那一種。英文名固定(考試用),說明可切中/英。

現在:雙語(中+EN)
💡 這頁補兩個盲點:① 英文名——考試&術語都英文,名字固定顯示、可切 English-only 模擬考;② 策略選單——每個情境列出所有有名字的做法,不是只有你慣用的那個。架構師的價值 = 知道整排角度並選對。

A. Agent / Workflow 編排

情境:一個任務單次 call 做不完 · a task that can't be done in one call

Strategy（英文名）何時用
Prompt Chaining串接拆成固定序列步驟,各步聚焦一件事;破解 constraint-heavy 長 prompt。split into fixed sequential steps, each focused; fixes constraint-heavy prompts.
Parallelization平行化一任務拆多個同時跑的子任務,再 aggregate(如逐材料評估)。split into simultaneous subtasks, then aggregate (e.g. evaluate each material).
Routing路由先分類 input → 導向專門 pipeline(不同主題不同 prompt/tool)。classify input first → route to a specialized pipeline.
Evaluator-Optimizer評估-優化producer 產出 → evaluator 評 → 不過就迴圈改,直到通過。produce → evaluate → loop back to improve until it passes.
Orchestrator-Workers協調者-工人coordinator 動態分派給 subagent(hub-and-spoke),自己彙整。coordinator dynamically delegates to subagents (hub-and-spoke) and synthesizes.
Agent (autonomous)自主代理步驟未知時,給抽象工具讓它自己擬計畫組合。when steps are unknown, give abstract tools and let it plan dynamically.
🔑 選型鐵則:步驟已知 → Workflow(可靠、可測);步驟未知才 → Agent。先可靠,再彈性。🔑 Rule: known steps → Workflow (reliable, testable); unknown steps → Agent. Reliability first.

B. Multi-Agent 協調模式（Anthropic 5 patterns）

情境:單一 agent 不夠,要拆多個 · when one agent isn't enough

Strategy何時用
Generator-Verifier生成-驗證品質關鍵輸出;verifier 只找錯、退回不修,且用 clean context。ROI 最高。quality-critical; verifier only finds faults & rejects (never fixes), clean context. Highest ROI.
Orchestrator-Subagent協調-子代理預設起手式;主 agent 做主線,只把 context-heavy 側查丟 subagent。default starting point; main agent works, offloads context-heavy sidequests.
Agent Teams代理團隊長期、多步、獨立的工作(大遷移);worker 常駐累積 context。long-running, multi-step independent work; workers persist.
Message Bus訊息匯流排事件驅動 pipeline(安全告警);難 debug,要紀律 logging。event-driven pipelines; hard to debug, needs disciplined logging.
Shared State共享狀態研究綜合,任一發現要即時給其他 agent;要明確終止條件。research-synthesis where any discovery must be visible to all; needs termination conditions.
⚠️ 鐵則:split by context, NOT by problem type。寫功能的 agent 也該寫它的測試(已有 context)。寫入動作單線程,其他 agent 只給 judgment。⚠️ Rule: split by context, not by problem type. Writes stay single-threaded; other agents only advise.

C. Prompt Engineering 技巧

情境:提升輸出品質/一致性 · improve output quality

Strategy何時用
Zero-shot零範例直接下指令,不給範例。instruction only, no examples.
One-shot / Few-shot (Multi-shot)單/多範例給 sample input/output 配對引導;corner case、格式一致最有效。give sample input/output pairs; best for corner cases & consistent formatting.
Chain-of-Thought (CoT)思維鏈要它逐步推理再答;複雜推理題。make it reason step-by-step before answering; complex reasoning.
Being Clear & Direct清楚直接第一行用 action verb 直指任務(Write/Identify/Generate)。first line = action verb stating the exact task.
Being Specific (Attributes / Steps)具體:屬性/步驟Type A 列輸出品質;Type B 給推理步驟。幾乎每個 prompt 都該有 A。Type A lists output qualities; Type B gives reasoning steps. Almost every prompt needs A.
XML StructuringXML 標籤用描述性標籤(<sales_records>)分隔插入的內容區塊。wrap injected content in descriptive tags to delineate sections.
Assistant Prefill預填助手訊息尾端塞 assistant 開頭,引導它從那接續(拿乾淨輸出)。seed an assistant opening so it continues from there (clean output).
Stop Sequences停止序列生成到指定字串立即停(該字串不含在輸出)。halt generation at a given string (string excluded from output).

D. Structured Output 結構化輸出

情境:要乾淨可解析的 JSON/資料 · need clean parseable data

Strategy何時用
Prefill + Stop Sequence預填+停止prefill ```json + stop ```。快、簡單,適合輕量擷取。prefill ```json + stop ```. Fast/simple, for lightweight extraction.
Tool Use + JSON Schema工具+JSON schema最可靠、消語法錯;但只防語法不防語意。複雜/高可靠用。most reliable, eliminates syntax errors (not semantic); for complex/reliable needs.
tool_choice: auto / any / forced工具選擇auto=可回文字;any=必呼叫某工具;forced=指定工具先跑。auto=may return text; any=must call a tool; forced=call a named tool.

E. Evaluation 評估

情境:證明 prompt/系統夠好 · prove it's good enough

Strategy何時用
Code Grader程式評分程式檢查:語法 valid、長度、關鍵字、可讀性。客觀但淺。programmatic checks: syntax valid, length, keywords. Objective but shallow.
Model Grader模型評分另一個 AI 模型評分;要它給 strengths/weaknesses/reasoning+score。最有彈性。another AI model grades; ask for strengths/weaknesses/reasoning+score. Most flexible.
Human Grader人工評分人審;最彈性但耗時。a person reviews; most flexible but slow.
A/B TestingA/B 測試兩版 prompt 用一個 metric 比,用資料決定。compare two prompt versions on one metric; let data decide.

F. RAG 檢索策略

情境:大文件查特定資訊 · query large docs

Strategy何時用
Size-based Chunking等長切塊等長切;最常用,配 overlap 保上下文。equal-length chunks; most common, add overlap for context.
Structure-based Chunking結構切塊依標題/段落切;需結構化文件(markdown/HTML)。split by headers/sections; needs structured docs.
Semantic Chunking語意切塊NLP 依語意分組;最進階、最複雜。NLP groups by meaning; most advanced.
Semantic Search (embeddings)語意搜尋向量 cosine 相似;懂語意但漏精確字。vector cosine similarity; understands meaning, misses exact terms.
BM25 (Lexical)關鍵字搜尋精確字匹配;稀有字權重高。補語意搜尋。exact term matching; rare terms weighted high. Complements semantic.
Hybrid + RRF混合+RRF語意+BM25 並行,用 Reciprocal Rank Fusion 合併。semantic + BM25 in parallel, merged via Reciprocal Rank Fusion.
Reranking重排序檢索後用 LLM 依相關性重排;增準度、加延遲。LLM reorders results by relevance after retrieval; more accurate, more latency.
Contextual Retrieval情境化檢索embed 前先給每個 chunk 加原文情境,防失去上下文。add situating context to each chunk before embedding.

G. Context 管理 / 可靠性

情境:省 context、控成本、防漂移 · save context, control cost

Strategy何時用
Prompt Caching提示快取重複的 system/tool 前綴設 cache breakpoint(≥1024 token,1hr)。cache breakpoints on repeated system/tool prefixes (≥1024 tokens, 1hr).
Progressive Disclosure漸進揭露主檔<500 行,細節拆附屬檔,需要才載(skills)。keep main file <500 lines; load detail files only when needed.
Subagent Isolation子代理隔離冗長探索丟 subagent、只回摘要,保主 context 乾淨。offload verbose work to a subagent, return only a summary.
Extended Thinking延伸思考複雜題開推理預算(≥1024,max_tokens 要更大);prompt 調不動才用。reasoning budget for hard tasks (≥1024, max_tokens larger); use after prompt tuning fails.

H. Claude Code 客製化（選對工具）

情境:要客製 Claude Code 行為 · customize Claude Code

Feature何時用
CLAUDE.mdalways-on:每個對話都載。專案標準/約束/風格。always-on, loads every conversation. Project standards/constraints.
Skillsrequest-driven:匹配才自動載。任務專屬知識、省 context。request-driven, auto-loads on match. Task-specific expertise.
Subagents隔離 context 委派;要不同工具權限、要隔離。isolated-context delegation; different tool access, isolation.
Hooksevent-driven:存檔/工具呼叫時 fire。自動 side effect/驗證。event-driven, fires on save/tool call. Automated side effects.
MCP Servers外部工具/整合(不同類別)。external tools/integrations (a different category).
