Anthropic Claude 認證四張證的官方考域配分（全來自官方 exam guide PDF v1.0, 2026-07）。CCAR-P 為 Tom 目標，附細目 + 落差分析。

四張證對照

代碼名稱對象費用題數
CCAO-FAssociate – Foundations商業/行銷/PM/教育$9960

四張都是 120 分鐘、及格 720/1000、全英文禁翻譯、效期 12 月 + 免費續證、每 12 月最多重考 4 次（14/30/90 天）。黃底 = 該證權重最高域。

① CCAO-F · Associate（把 Claude 當生產力工具用）

會寫 prompt、評估輸出、選對產品/模型、把 Claude 整進工作流、配 Projects、負責任使用。無需寫程式。
考域配分
Output Evaluation and Validation（評估輸出、抓幻覺）21%
Workflow Integration and Solution Design16%
Governance, Risk, and Responsible Use15%
Prompting and Task Execution14%
Product and Model Selection12%
Configuration and Knowledge Management（Projects/連接器）12%
Troubleshooting and Optimization10%
重心在「評估與驗證輸出」，不是寫扣。適合先拿來熱身/當團隊入門證。

② CCDV-F · Developer（build/integrate/ship 生產級應用）

用 Claude Agent SDK 建 agent/workflow、透過 API 與 client SDK 整合、操作 Claude Code、建 MCP server。建議 Python/TypeScript + REST。
考域配分
Applications and Integration（API 機制/應用設計/軟工基礎）33.1%
Model Selection and Optimization（含 token/caching）16.8%
Agents and Workflows（Agent SDK/多 agent）14.7%
Prompt and Context Engineering11.0%
Tools and MCPs10.6%
Security and Safety8.1%
Claude Code3.1%
Eval, Testing, and Debugging2.6%
⚠️ Applications & Integration 一域就佔 1/3——這張證的成敗全看 API 整合與應用設計。

③ CCAR-F · Architect Foundations（設計 Claude agent 架構）

情境題導向（6 情境抽 4）：客服 agent、Claude Code、多 agent 研究、開發者生產力、CI/CD、結構化資料擷取。
考域配分
Agentic Architecture & Orchestration（agent loop/coordinator-subagent）27%
Claude Code Configuration & Workflows20%
Prompt Engineering & Structured Output20%
Tool Design & MCP Integration18%
Context Management & Reliability15%
= CCAR-P 的「技術核心版」，少了企業合規/利害關係人/lifecycle。對 Tom 幾乎全是盤點已知，可當 CCAR-P 前哨。

④ CCAR-P · Architect Professional（Tom 目標）

右側色塊 = Tom 落差自評：強 盤點已知 · 中 要補 · 缺 主攻。63 題 / 720 分過。

1. Solution Design & Architecture 17%強

  • 商業問題→Claude 方案、端到端架構、workflow/agentic/augmented-LLM 選型、多 agent 編排、decomposition、對齊商業價值支柱

2. Claude Models, Prompting & Context Eng 13%強

  • 模型 trade-off 選型、system prompt/guardrail、zero/few-shot/CoT、context window/token、prompt 重用（caching、Skills）

3. Integration 19%中 最重

  • capability bloat、authn/authz 安全缺口、accuracy-latency、大規模 observability、RAG chunking/indexing/retrieval、連線協議（MCP/API·CLI/agent-to-agent）、progressive vs monolithic context

4. Evaluation, Testing & Optimization 16%中

  • eval metrics、eval dataset/mixed methodology、A/B、診斷（prompt failure/hallucination/model mismatch）、token/latency/cost、observability

5. Governance, Safety & Risk 14%缺

  • guardrail、風險/failure mode、HITL、合規（GDPR/HIPAA/FedRAMP ← 見 法規卡）、倫理（bias/fairness/transparency）

6. Stakeholder Communication & Lifecycle 14%中

  • structured discovery、trade-off 溝通、SLA/期望、架構文件、lifecycle（discovery→design→handoff→monitor→iterate）

7. Developer Productivity & Operational Enablement 7%強

  • Claude 工具/環境配置（Claude Code）、AI 輔助 workflow、debug/維運

Tom 備考策略（集中火力）

7 域裡 6 域「強～中」，主攻 3 塊：
  • 缺 Domain 5 合規（GDPR/HIPAA/FedRAMP）→ 法規卡已備
  • 中 Domain 4 正式 eval：dataset/A/B/hallucination 歸因
  • 中 Domain 3 authz/observability 設計題
建議路徑：先考 CCAR-F（技術核心、幾乎已會）當前哨 → 再上 CCAR-P（多補合規+利害關係人）。官方全課程的 6 門必修可對齊用語。

報名路徑（官方考綱 §10）

  1. Anthropic Partner Academy 找對應證頁面，下載考綱 + 政策
  2. 結帳（partner tier 折扣自動套用）
  3. 建 Pearson VUE 帳號 → 排期
  4. 選線上監考或考場（24 小時前可改期）
