📝 錯題清單

D4Building API · Prompt Eval 隨堂
要為 prompt evaluation 產測試資料集，該用哪個模型來「生成」？
✅ 正解：用較快的模型（如 Haiku）
產 dataset 是機械性工作，用快/省的模型即可；把貴的模型留給「被測試的那個 prompt」本身。別選「最貴的模型」。
D4Building API · Prompt Eval 隨堂
eval workflow 中，用 Claude 產生了一批回應之後，下一步是什麼？
✅ 正解：把回應丟給 grader 評分
eval 流程順序：寫 prompt → 建 dataset → 產回應 → grade（評分） → 依分數迭代。產完回應不是去部署、也不是再加題，是先量分數。
D4Building API · Prompt Eval 隨堂
哪一種 grader 是「用另一個 AI 模型」來評估輸出品質？
✅ 正解：Model grader
三種 grader：Code grader=程式檢查(語法/長度/valid)；Model grader=另發一次 API 讓模型評分(最有彈性)；Human grader=人工。別跟 code/syntax grader 搞混。
D4Building API · Prompt Engineering 隨堂
「提供 sample input/output 配對來引導 AI 回應」是哪個 prompt 技巧？
✅ 正解：One-shot / Multi-shot prompting
給範例=shot：one-shot=1 個範例、multi-shot=多個。別跟 XML structuring(分隔內容區塊)或 iterative refinement(反覆修)搞混——關鍵字「sample input/output 配對」就是 shot。
