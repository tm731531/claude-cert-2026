D1 · ❗連錯 2 次Agents & Workflows · 隨堂
寫報告 → 檢查夠不夠好 → 不夠就改進。這是什麼 pattern？
✅ 正解：Evaluator-Optimizer
🟡 你最頑固的一題：producer 產出 → evaluator 評 → 不過就 loop 回去改。關鍵詞「檢查+改進迴圈」= Evaluator-Optimizer。別選 chaining(那是純序列、不回頭)。這個名字背死它。
D1綜合 quiz · Q21
為什麼 Environment Inspection（環境檢查）對 agent 至關重要？
✅ 正解：讓 agent 觀察並理解「自己動作的結果」
agent 無法預測點按鈕/改檔的結果 → 每個動作後要檢查環境(截圖、讀檔)才知進度、抓錯、適應。不是為了「更快/更省/簡化工具」。
✅ 已攻克(重考訂正)：Q3 不同內容→Routing;Q6 要可靠→Workflow;Q7 步驟已知→Workflow with predetermined steps。workflow-vs-agent 陷阱已過關。
D4Building API · Prompt Eval 隨堂
要為 prompt evaluation 產測試資料集，該用哪個模型來「生成」？
✅ 正解：用較快的模型（如 Haiku）
產 dataset 是機械性工作，用快/省的模型即可；把貴的模型留給「被測試的那個 prompt」本身。別選「最貴的模型」。
D4Building API · Prompt Eval 隨堂
eval workflow 中，用 Claude 產生了一批回應之後，下一步是什麼？
✅ 正解：把回應丟給 grader 評分
eval 流程順序：寫 prompt → 建 dataset → 產回應 → grade（評分） → 依分數迭代。產完回應不是去部署、也不是再加題，是先量分數。
D4 · ❗連錯 2 次Prompt Eval + 綜合 Q8
哪一種 grader 是「用另一個 AI 模型」來評估輸出品質？
✅ 正解：Model grader
三種 grader：Code grader=程式檢查(語法/長度/valid)；Model grader=另發一次 API 讓模型評分(最有彈性)；Human grader=人工。別跟 code/syntax grader 搞混。
D4Building API · Prompt Engineering 隨堂
「提供 sample input/output 配對來引導 AI 回應」是哪個 prompt 技巧？
✅ 正解：One-shot / Multi-shot prompting
給範例=shot：one-shot=1 個範例、multi-shot=多個。別跟 XML structuring(分隔內容區塊)或 iterative refinement(反覆修)搞混——關鍵字「sample input/output 配對」就是 shot。
D2Building API · Tool Use 隨堂
batch tool 解決什麼問題？
✅ 正解：減少多工具時的來回通訊次數（單一 request-response，取代逐次呼叫）
陷阱是「讓工具跑更快」——不是加速單一工具,而是把多個 tool 呼叫塞進一輪,省掉多次往返。關鍵字：parallel / 單一回合。
✅ MCP D2 已攻克（2026-07-12 重測 7/7 全對）：下面 3 題(Inspector / stdio / Prompts)重測都對了。尤其 Prompts 用「user 控制 workflow」框架答對——這 3 題保留當紀錄，不再是弱點。
D2 · ✅已訂正MCP 課 · 隨堂
建好 MCP server，想在接 Claude 前測試工具，最好的方式？
✅ 正解：用瀏覽器裡的 MCP Inspector（mcp dev server.py）
Inspector = 不接 app 就能手動測工具的 in-browser debugger。別選「直接接 Claude」或「production 測」。
D2MCP 課 · 隨堂
MCP server 與 client 在開發時，最常用什麼方式連線？
✅ 正解：同機的 standard input/output（stdio）
MCP transport 無關(stdio / HTTP / WebSocket 都行),但開發時最常見=同機 stdio。不是走 database / internet / email。
D2MCP 課 · 隨堂
想給用戶一個「高品質、預先測過」的格式化文件指令，該用哪個 MCP 功能？
✅ 正解：Prompts
MCP 三件套別搞混：Tools=執行動作、Resources=曝資料給讀、Prompts=server 端預寫測過的指令模板(client 端變 slash command)。「預先測過的指令」→ Prompts。

CCDV-FBuilding API · 進階功能隨堂
Citations 的主要用途是什麼？
✅ 正解：建立從 Claude 回應追回原文特定段落的軌跡（來源溯源/透明度）
不是壓縮文件、不是數字數、不是自動生註腳。但這是 CCDV-F 功能，CCAR-F 不考，略過即可。
CCDV-FBuilding API · 進階功能隨堂
Code Execution 工具的關鍵限制是什麼？
✅ 正解：無網路存取，跑在隔離的 Docker container
資料進出靠 Files API。同樣是 CCDV-F 功能,CCAR-F 不考。
